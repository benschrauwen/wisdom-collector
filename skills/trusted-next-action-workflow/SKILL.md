# Trusted Next-Action Workflow

Use this when work feels mentally noisy, inbox-heavy, or stuck in vague shoulds and unfinished commitments.

## Core principle
Keep open loops out of working memory and inside a trusted external system. The goal is not more lists; it is a mind that can stop rehearsing unfinished commitments.

## Five-step loop
1. **Capture** everything that has your attention.
2. **Clarify** what each item means and whether it is actionable.
3. **Organize** the result into the right reminder container.
4. **Reflect** regularly so the system stays current and trustworthy.
5. **Engage** by choosing the best next action in the moment.

## Process items one at a time
When emptying an inbox or doing a mind sweep:
- take the **top item first**
- handle **one item at a time**
- **never put items back in**
- finish the decision before moving on

Ask, in order:
- **What is it?**
- **Is it actionable?**
- **What is the next physical, visible action?**

If you cannot state the next physical action, the item is not clarified enough yet.

## Decide the fate of each item
### If it is actionable
- **Do it now** if the next action takes **2 minutes or less**.
- **Delegate it** if someone else is the right doer.
- **Defer it** if you are the right person but will do it later.

### If it is not actionable
- **Trash** it if it has no value.
- **Incubate** it if it may matter later.
- **Reference** it if it is support material you may need again.

## Keep reminder categories separate
- **Projects**: any desired outcome requiring more than one action step.
- **Next Actions**: single physical actions that are actionable but not tied to a specific day/time.
- **Waiting For**: items delegated or pending someone else’s response.
- **Calendar**: only hard landscape — day-specific or time-specific commitments and reminders that must appear on a particular date.
- **Someday/Maybe**: items you might want later, not now.
- **Reference**: nonactionable information.
- **Project support material**: notes, plans, and files for active projects; keep them separate from reminders.

Rule of thumb: if an item still feels vague, it needs more clarification before it can be organized.

## Choose the next action intelligently
Check in this order:
1. **Context** — what can you do where you are?
2. **Time** — what fits in the time window?
3. **Energy** — what matches your current horsepower?
4. **Priority** — which remaining option matters most?

Do not use priority to replace clarity. Priority only helps after the work is organized.

## Keep the system trustworthy
A list only relieves stress if it is current.
- Review the **calendar** frequently.
- Review **Next Actions** when you have discretionary time.
- Review **Projects**, **Waiting For**, and **Someday/Maybe** often enough that you do not need to remember them.
- Do a **Weekly Review** to get clear, current, and complete again.

If the system slips, reset it:
1. mind sweep
2. clean the lists
3. restore missing projects and next actions
4. recover anything that leaked outside the system
5. re-establish trust through review

## Plan fuzzy projects
For any project that still feels fuzzy, work in this order:
1. **Purpose / principles** — why does it matter, and what boundaries apply?
2. **Outcome vision** — what does success look like?
3. **Brainstorming** — generate ideas without judging them.
4. **Organizing** — sort by component, sequence, and priority.
5. **Next actions** — define the very next physical step for each moving part.

If a project is still on your mind, it usually needs either a clearer outcome, a next action, or both.

## Practical guardrails
- Do not try to manage capture, processing, organizing, reviewing, and doing all at once.
- Do not let calendars become wishful to-do lists.
- Do not let inboxes become storage.
- Keep tools simple, reachable, and pleasant enough to use consistently.
- Favor external reminders over mental juggling.

## Quick prompts
- What does done look like?
- What is the next physical action?
- Who has it?
- What belongs on the calendar, and what belongs on an action list?
- What open loop is still living only in memory?
